# Agent guidance

## Documentation expectations

The `docs/` directory is the primary place for documentation about the homelab.

It should contain documentation about topics such as:

- homelab architecture
- system design
- tooling and workflows
- image build and deployment behavior
- operational practices
- rationale for current decisions
- future plans and intended changes

When working in this repository, consult the `docs/` directory to understand:

- how the system currently works
- why things are implemented the way they are
- what assumptions and constraints exist
- where the project is intended to go next

## Maintaining documentation

Documentation should be kept accurate to the current state of the system.

Whenever changes are made to the code, configuration, workflows, architecture, or operational model, the relevant documentation in `docs/` should also be updated so that it continues to reflect reality.

In short:

- use `docs/` to understand the system
- update `docs/` when changing the system
